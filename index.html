VLANs - Virtual LANs

Simple Definition

VLANs create separate broadcast domains on the same physical switch, like having multiple virtual switches in one box.

VLAN Types
  • Default VLAN: VLAN 1 - All ports start here
  • Data VLAN: Carries user data
  • Voice VLAN: Prioritizes VoIP traffic
  • Management VLAN: For switch management
  • Native VLAN: For untagged trunk traffic
Port Types
  • Access - One VLAN only
  • Trunk - Multiple VLANs
  • Hybrid - Both (rare)
Essential Configuration
! Create VLAN 10 named SALES
Switch(config)# vlan 10
Switch(config-vlan)# name SALES

! Assign port to VLAN 10
Switch(config)# interface FastEthernet0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

! Configure trunk port
Switch(config)# interface GigabitEthernet0/1
Switch(config-if)# switchport mode trunk
Common Mistakes

1. Forgetting to set port mode before assigning VLAN
2. Mismatched native VLAN on trunk links
3. Not creating VLAN on both switches

Inter-VLAN Routing

Real-World Analogy

A router with subinterfaces is like a building concierge with different mail slots for each floor (VLAN). They sort and deliver packages between floors.

Hierarchical Design
Core Layer - Fast Backbone
Distribution Layer - Policy & Routing
Access Layer - End Devices
Router-on-a-Stick Configuration
! Enable physical interface
Router(config)# interface GigabitEthernet0/0
Router(config-if)# no shutdown

! Create subinterface for VLAN 10
Router(config)# interface GigabitEthernet0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0

! Create subinterface for VLAN 20
Router(config)# interface GigabitEthernet0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0
Exam Tip

Always check: 1) Physical interface is up, 2) Subinterface encapsulation matches VLAN ID, 3) IP addresses are in different subnets!

Spanning Tree Protocol (STP)

What Problem Does STP Solve?

Without STP: Broadcast storm! Frames loop forever, network crashes. With STP: One active path, backup paths blocked but ready.

Port States Timeline
Blocking (20s) Listens only
Listening (15s) Sends/recieves BPDUs
Learning (15s) Builds MAC table
Forwarding Sends data
Bridge Election Order
  1. Lowest Bridge Priority (default 32768)
  2. Lowest MAC Address

Memory Trick: "**P**riority then **M**AC" = "**P**ass **M**y Exam!"

Critical STP Commands
! Make switch root for VLAN 10
Switch(config)# spanning-tree vlan 10 root primary

! Configure PortFast on access ports
Switch(config)# interface FastEthernet0/1
Switch(config-if)# spanning-tree portfast

! Configure EtherChannel (link aggregation)
Switch(config)# interface range FastEthernet0/1-2
Switch(config-if-range)# channel-group 1 mode active

Subnetting & VLSM

The Pizza Method

Regular subnetting: Cut pizza into equal slices. VLSM: Cut into different sizes - big slice for department (60 hosts), small slice for WAN link (2 hosts).

Magic Number Method

Step 1: 256 - subnet mask value
Step 2: That's your block size!
Example: /26 = 255.255.255.192
256 - 192 = 64
Blocks: .0, .64, .128, .192

Practice Quiz

Question 1: Which command assigns VLAN 20 to an interface?
Correct: switchport access vlan 20
Question 2: What is the native VLAN by default?
Correct: VLAN 1
Question 3: How many usable hosts in a /29 subnet?
Calculation: /29 = 255.255.255.248
256 - 248 = 8 (block size)
Usable hosts = 8 - 2 = 6 hosts
